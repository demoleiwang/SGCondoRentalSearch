## Context

- 63 岁，退休中学华文老师，先生 65 岁，在家具厂做半职。
- 两人合计月入 + CPF 分期 ~SGD 4,800。没有大开销，目标是"住得体面、不吃力"。
- 现在住在女儿给她买的 Bishan HDB，准备搬到自己租的更小单位，把那套还给女儿。
- 目标：2BR HDB 整租或小 condo，$2,500–$3,300/月，Ang Mo Kio / Bishan / Yishun 都行。
- 非常看重数字可靠性——退休前教语文兼带过社科统计课，会自己做 Excel 对比。
- 对互联网应用怀疑，喜欢"数据来源是哪、样本量多少、更新到什么时候"这种元信息。

## Search Style

- 纯中文，用词正式："在 Ang Mo Kio 找 2 房租金不超过 3000 的 HDB"。
- 会追问："数据是 URA 官方吗？""是中位数还是平均数？""样本量够不够？"（虽然网页 UI 不能对话，她会用工具里的每一条文本当 "证据"）。
- 打开任何图表都会先放大看坐标轴和单位。
- 每搜 1 次会记笔记对比 3-4 个项目，数据要能稳定复现。

## Success Criteria

1. 数据来源、更新时间、样本量都能看到（URA / HDB / 季度 / 合约笔数）。
2. 结果页上每一条数字都能自圆其说：为什么 $3,120？基于哪个季度、多少笔合约？
3. 中文 UI 足够完整，不是把英文界面翻一半。
4. HDB 组屋结果的租金是真实成交价，不是估算。

## Quit Triggers

- 数字对不上（比如中位数 $3,200 但 P25–P75 是 $2,000–$4,000，显然样本太少）却没有警示。
- 英文术语塞在中文界面里（"rental contracts"、"psf" 直接裸露不翻译）。
- 图表坐标轴没单位或单位不一致（$/psf 和 $/月混在一张图）。
- 某些项目只有 1–2 笔合约还当成"中位数"来展示——在她眼里这就是数据造假。
